# Custom fields

Custom fields allow you to add specific information to your contacts beyond the standard fields.

## Why use custom fields?

### Advanced personalization

Use custom variables in your emails:
- `{{sector}}` → "Fintech"
- `{{companySize}}` → "50-100 employees"
- `{{contactSource}}` → "LinkedIn Sales Navigator"

### Segmentation

Filter and segment your contacts according to your business criteria.

### Reporting

Analyze your performance according to custom dimensions.

## Create a custom field

### 1. Access settings

1. Go to **Settings**
2. Click on **Custom fields**
3. Click on **+ New field**

### 2. Configure the field

| Parameter | Description | Example |
|-----------|-------------|---------|
| **Name** | Technical identifier (no spaces) | `industry_sector` |
| **Label** | Display name | "Industry Sector" |
| **Type** | Data type | Text, Number, Date, List |
| **Required** | Required field? | No |
| **Default value** | Value if not provided | - |

### 3. Available field types

| Type | Description | Use case |
|------|-------------|----------|
| **Text** | Free text string | Note, comment |
| **Number** | Numeric value | Revenue, employee count |
| **Date** | Date (day/month/year) | First contact date |
| **Dropdown** | Choice among defined options | Sector, source |
| **Checkbox** | Yes/No | GDPR accepted |
| **URL** | Web link | LinkedIn profile |

## Useful field examples

| Field | Type | Variable |

## Using custom fields

### In forms

Custom fields automatically appear in:
- The contact creation form
- The contact edit form
- CSV import (column mapping)

### In emails

Use the `{{field_name}}` syntax in your templates:

```
Hello {{firstName}},

As a {{sector}} expert, you know that...

With a team of {{companySize}} people,
you certainly need...
```

### In filters

1. Go to **Contacts**
2. Click on **Filters**
3. Select the custom field
4. Define the condition (equals, contains, etc.)

## Import with custom fields

### CSV format

Name your columns with the technical name of the field:

```csv
email,firstName,lastName,industry_sector,company_size
john@example.com,John,Smith,Fintech,50-100
mary@example.com,Mary,Johnson,SaaS,10-50
```

### Mapping

During import, FluenzR automatically detects columns corresponding to your custom fields.

## Edit a custom field

> **Warning**
Editing a field can impact existing data. Proceed with caution.

### What you can edit

- Label (display name)
- Default value
- Dropdown options (add only)

### What you cannot edit

- Technical name (identifier)
- Field type

To change the type, create a new field and migrate the data.

## Delete a custom field

1. Go to **Settings** > **Custom fields**
2. Click on the delete icon
3. Confirm

> **Danger**
Deletion is irreversible. All values for this field will be lost for all contacts.

## Best practices

### Naming

| Recommended | Avoid |
|-------------|-------|
| `industry_sector` | `Industry Sector` |
| `company_size` | `size` |
| `first_contact_date` | `date1` |

### Organization

- Limit the number of fields (10-15 max)
- Prefer lists over free text fields
- Document the use of each field
- Regularly review unused fields

### Default values

- Use relevant default values
- "Not specified" rather than empty for lists
- Facilitate filters with consistent values
