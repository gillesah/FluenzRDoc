# Créer un contact

Apprenez à ajouter des contacts un par un dans FluenzR.

## Création manuelle

### 1. Accéder au formulaire

1. Dans le menu latéral, cliquez sur **Contacts**
2. Cliquez sur le bouton **+ Ajouter un contact**

### 2. Remplir les informations

#### Informations obligatoires

| Champ | Description |
|-------|-------------|
| **Email** | L'adresse email du contact (doit être unique) |

#### Informations optionnelles

| Champ | Description | Variable |
|-------|-------------|----------|
| Prénom | Prénom du contact | `{{firstName}}` |
| Nom | Nom de famille | `{{lastName}}` |
| Entreprise | Nom de l'entreprise | `{{company}}` |
| Poste | Fonction/titre | `{{jobTitle}}` |
| Téléphone | Numéro de téléphone | `{{phone}}` |
| Site web | URL du site | `{{website}}` |

### 3. Ajouter à un groupe (optionnel)

Sélectionnez un ou plusieurs groupes pour organiser votre contact dès sa création.

### 4. Add tags (optional)

Add tags to make filtering easier later.

### 5. Document consent (required)

FluenzR requires you to document the consent source for each contact. This is a legal requirement (GDPR, CAN-SPAM):

| Consent Source | Description |
|----------------|-------------|
| **Web Opt-in** | Contact registered via a form on your website |
| **Event** | Contact obtained at an event (trade show, conference) |
| **Business Card** | Contact obtained through business card exchange |
| **Social Network** | Contact obtained via LinkedIn or other professional network |
| **Referral** | Contact recommended by a third party |
| **Existing** | Contact already in your database (migration) |
| **Other** | Other source to specify in notes |
| **Not Specified** | Avoid - may cause compliance issues |

You can also add **consent notes** to document the collection circumstances more precisely.

> **Warning**
> Contacts without documented consent source may be excluded from certain mailings for compliance reasons.

### 6. Save

Click **Save** to create the contact.

## Champs personnalisés

Si vous avez créé des [champs personnalisés](/docs/contacts/custom-fields), ils apparaissent dans le formulaire de création.

Exemple :
- Secteur d'activité
- Taille de l'entreprise
- Source de la lead

## Validation de l'email

FluenzR vérifie automatiquement :

- **Format** : L'email doit être valide (format@domaine.extension)
- **Unicité** : L'email ne doit pas déjà exister dans votre base

> **⚠️ Attention**
Si l'email existe déjà, vous recevrez un message d'erreur. Modifiez le contact existant ou utilisez un autre email.

## Bonnes pratiques

### Qualité des données

- Remplissez un maximum de champs pour une meilleure personnalisation
- Vérifiez l'orthographe du prénom (majuscule initiale)
- Utilisez le format standard pour les entreprises

### Organisation

- Assignez toujours à un groupe pertinent
- Utilisez des tags cohérents avec votre stratégie
- Complétez le pipeline si c'est un prospect qualifié

### GDPR & Compliance

- **Always document the consent source** (required in FluenzR)
- The consent date is automatically recorded
- Add notes if the source needs clarification
- Only contact people who have given their consent
- For CAN-SPAM compliance (USA), configure your physical address in settings

## Après la création

Une fois le contact créé, vous pouvez :

- **L'ajouter à une campagne** : Depuis la page du contact ou lors de la configuration de la campagne
- **Voir son historique** : Emails envoyés, ouvertures, clics, réponses
- **Modifier ses informations** : Cliquez sur le contact pour éditer sa fiche
- **Le qualifier** : Positionnez-le dans le pipeline

## Création rapide depuis une campagne

Vous pouvez aussi créer des contacts directement depuis l'écran d'ajout de contacts d'une campagne :

1. Ouvrez une campagne
2. Cliquez sur **Ajouter des contacts**
3. Cliquez sur **Créer un nouveau contact**
4. Remplissez le formulaire rapide
5. Le contact est ajouté à la campagne automatiquement
