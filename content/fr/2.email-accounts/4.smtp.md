# Configuration SMTP

Pour les fournisseurs email autres que Gmail et Outlook, FluenzR supporte la connexion via SMTP.

## Quand utiliser SMTP ?

- Serveur email auto-hébergé
- Fournisseur email français (OVH, Gandi, etc.)
- Service email professionnel (Zoho, ProtonMail Bridge, etc.)
- Besoins spécifiques de configuration

> **⚠️ Attention**
La connexion SMTP stocke vos identifiants de manière chiffrée (AES-256). Pour une sécurité maximale, préférez OAuth (Gmail/Outlook) quand c'est possible.

## Informations requises

Pour configurer SMTP, vous aurez besoin de :

| Paramètre | Description | Exemple |
|-----------|-------------|---------|
| **Serveur SMTP** | Adresse du serveur sortant | smtp.example.com |
| **Port** | Port de connexion | 587 (TLS) ou 465 (SSL) |
| **Nom d'utilisateur** | Généralement votre email | user@example.com |
| **Mot de passe** | Mot de passe ou app password | ********** |
| **Chiffrement** | Type de sécurité | TLS (recommandé) |

## Configuration pas à pas

### 1. Accéder à la configuration

1. Allez dans **Comptes Email**
2. Cliquez sur **+ SMTP**

### 2. Remplir les informations

1. **Email** : Votre adresse email complète
2. **Nom d'affichage** : Le nom qui apparaîtra comme expéditeur
3. **Serveur SMTP** : L'adresse de votre serveur
4. **Port** : Généralement 587 (TLS) ou 465 (SSL)
5. **Nom d'utilisateur** : Souvent identique à l'email
6. **Mot de passe** : Votre mot de passe (ou app password)
7. **Chiffrement** : Sélectionnez TLS ou SSL

### 3. Tester la connexion

Cliquez sur **Tester** pour vérifier que la configuration fonctionne. Un email de test sera envoyé à votre propre adresse.

### 4. Configurer IMAP (optionnel)

Pour la détection des réponses, configurez également IMAP :

| Paramètre | Description | Exemple |
|-----------|-------------|---------|
| **Serveur IMAP** | Adresse du serveur entrant | imap.example.com |
| **Port** | Port de connexion | 993 (SSL) |

## Configurations courantes

### OVH Mail

```
SMTP: ssl0.ovh.net
Port: 587 (TLS)
IMAP: ssl0.ovh.net
Port IMAP: 993 (SSL)
```

### Gandi

```
SMTP: mail.gandi.net
Port: 587 (TLS)
IMAP: mail.gandi.net
Port IMAP: 993 (SSL)
```

### Zoho Mail

```
SMTP: smtp.zoho.eu (ou smtp.zoho.com)
Port: 587 (TLS)
IMAP: imap.zoho.eu
Port IMAP: 993 (SSL)
```

> **ℹ️ Info**
Pour Zoho, vous devrez générer un "App Password" dans les paramètres de sécurité de votre compte.

### Infomaniak

```
SMTP: mail.infomaniak.com
Port: 587 (TLS)
IMAP: mail.infomaniak.com
Port IMAP: 993 (SSL)
```

### ProtonMail (via Bridge)

```
SMTP: 127.0.0.1
Port: 1025
IMAP: 127.0.0.1
Port IMAP: 1143
```

> **⚠️ Attention**
ProtonMail nécessite l'application Bridge installée et active sur votre machine.

## App Passwords

Certains fournisseurs nécessitent un "mot de passe d'application" au lieu de votre mot de passe principal :

### Quand utiliser un App Password ?

- Authentification à deux facteurs (2FA) activée
- Exigence de sécurité du fournisseur
- Politique de l'organisation

### Comment le générer ?

1. Connectez-vous à l'interface web de votre fournisseur
2. Allez dans Paramètres > Sécurité
3. Cherchez "Mots de passe d'application" ou "App Passwords"
4. Créez un nouveau mot de passe pour "FluenzR"
5. Copiez le mot de passe généré dans FluenzR

## Résolution de problèmes

### "Connexion refusée"

- Vérifiez le port et le serveur
- Assurez-vous que votre firewall n'est pas bloquant
- Testez avec un autre port (465 au lieu de 587)

### "Authentification échouée"

- Vérifiez le nom d'utilisateur (souvent l'email complet)
- Utilisez un App Password si 2FA est activé
- Vérifiez les majuscules/minuscules du mot de passe

### "Certificat invalide"

- Assurez-vous d'utiliser le bon type de chiffrement
- Contactez votre fournisseur si le certificat est expiré

### "Limite d'envoi atteinte"

Les serveurs SMTP ont souvent des limites :
- Vérifiez les limites de votre fournisseur
- Espacez vos envois
- Contactez votre hébergeur pour augmenter les quotas

## Sécurité

### Stockage des identifiants

- Les identifiants SMTP sont chiffrés en AES-256
- Jamais stockés en clair dans la base de données
- Accessibles uniquement par le système d'envoi

### Recommandations

1. Utilisez un App Password dédié
2. Activez les alertes de connexion chez votre fournisseur
3. Changez régulièrement le mot de passe
4. Préférez OAuth quand c'est possible
