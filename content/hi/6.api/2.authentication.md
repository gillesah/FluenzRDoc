# API प्रमाणीकरण

FluenzR API प्रमाणीकरण के लिए API कुंजी का उपयोग करता है।

## API कुंजी प्राप्त करें

### आवश्यकताएं

- सक्रिय **Pro** योजना
- खाते पर प्रशासक अधिकार

### कुंजी उत्पन्न करें

1. FluenzR में लॉग इन करें
2. **सेटिंग्स** > **API** पर जाएं
3. **नई कुंजी उत्पन्न करें** पर क्लिक करें
4. कुंजी को एक वर्णनात्मक नाम दें
5. प्रदर्शित कुंजी को तुरंत कॉपी करें

> **चेतावनी**
कुंजी केवल एक बार दिखाई जाती है। इसे सुरक्षित स्थान पर रखें। यदि आप इसे खो देते हैं, तो आपको एक नई कुंजी उत्पन्न करनी होगी।

## API कुंजी का उपयोग करें

### प्राधिकरण हेडर

प्रत्येक अनुरोध के \`Authorization\` हेडर में अपनी API कुंजी शामिल करें:

\`\`\`bash
curl -X GET "https://api.fluenzr.co/v1/contacts" \\
  -H "Authorization: Bearer flz_sk_your_api_key_here" \\
  -H "Content-Type: application/json"
\`\`\`

### कुंजी प्रारूप

FluenzR API कुंजी इस प्रारूप का पालन करती हैं:

\`\`\`
flz_sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
\`\`\`

- \`flz\`: FluenzR उपसर्ग
- \`sk\`: गुप्त कुंजी
- \`xxx...\`: 32 अल्फ़ान्यूमेरिक वर्ण

## भाषा के अनुसार उदाहरण

### cURL

\`\`\`bash
curl -X GET "https://api.fluenzr.co/v1/contacts" \\
  -H "Authorization: Bearer flz_sk_abc123..." \\
  -H "Content-Type: application/json"
\`\`\`

### JavaScript (fetch)

\`\`\`javascript
const response = await fetch('https://api.fluenzr.co/v1/contacts', {
  headers: {
    'Authorization': 'Bearer flz_sk_abc123...',
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
\`\`\`

### JavaScript (axios)

\`\`\`javascript
import axios from 'axios';

const client = axios.create({
  baseURL: 'https://api.fluenzr.co/v1',
  headers: {
    'Authorization': 'Bearer flz_sk_abc123...'
  }
});

const { data } = await client.get('/contacts');
\`\`\`

### Python (requests)

\`\`\`python
import requests

headers = {
    'Authorization': 'Bearer flz_sk_abc123...',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.fluenzr.co/v1/contacts',
    headers=headers
)

data = response.json()
\`\`\`

### PHP

\`\`\`php
<?php
$ch = curl_init();

curl_setopt_array($ch, [
    CURLOPT_URL => 'https://api.fluenzr.co/v1/contacts',
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => [
        'Authorization: Bearer flz_sk_abc123...',
        'Content-Type: application/json'
    ]
]);

$response = curl_exec($ch);
$data = json_decode($response, true);
\`\`\`

### Ruby

\`\`\`ruby
require 'net/http'
require 'json'

uri = URI('https://api.fluenzr.co/v1/contacts')
http = Net::HTTP.new(uri.host, uri.port)
http.use_ssl = true

request = Net::HTTP::Get.new(uri)
request['Authorization'] = 'Bearer flz_sk_abc123...'
request['Content-Type'] = 'application/json'

response = http.request(request)
data = JSON.parse(response.body)
\`\`\`

## अपनी API कुंजियों का प्रबंधन करें

### मौजूदा कुंजियां देखें

**सेटिंग्स** > **API** में, अपनी कुंजी सूची देखें:

| नाम | बनाया गया | अंतिम उपयोग | कार्रवाइयां |
|------|---------|-----------|---------|
| Production | 01/01/2025 | 01/10/2025 | रद्द करें |
| Dev/Test | 01/05/2025 | 01/08/2025 | रद्द करें |

### कुंजी रद्द करें

1. कुंजी के आगे **रद्द करें** पर क्लिक करें
2. रद्दीकरण की पुष्टि करें

> **खतरा**
रद्दीकरण तत्काल और अपरिवर्तनीय है। इस कुंजी का उपयोग करने वाले सभी एप्लिकेशन काम करना बंद कर देंगे।

### सर्वोत्तम अभ्यास

#### प्रति परिवेश एक कुंजी

| परिवेश | कुंजी | अनुमतियां |
|-------------|-----|-------------|
| Production | flz_sk_prod_xxx | सभी |
| Staging | flz_sk_stag_xxx | सभी |
| Development | flz_sk_dev_xxx | केवल पढ़ने के लिए |

#### कभी भी कुंजियों को उजागर न करें

**कभी न करें:**
- Git रेपो में कमिट करें
- क्लाइंट-साइड शामिल करें (frontend)
- ईमेल/चैट के माध्यम से साझा करें

**हमेशा करें:**
- पर्यावरण चर का उपयोग करें
- secrets manager में स्टोर करें
- केवल सर्वर-साइड रखें

#### पर्यावरण चर

\`\`\`bash
# .env (कमिट न करें)
FLUENZR_API_KEY=flz_sk_abc123...
\`\`\`

\`\`\`javascript
// Node.js
const apiKey = process.env.FLUENZR_API_KEY;
\`\`\`

\`\`\`python
# Python
import os
api_key = os.environ.get('FLUENZR_API_KEY')
\`\`\`

## प्रमाणीकरण त्रुटियां

### 401 Unauthorized

\`\`\`json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or missing API key"
  }
}
\`\`\`

**संभावित कारण:**
- API कुंजी गायब
- अमान्य API कुंजी
- रद्द की गई API कुंजी

### 403 Forbidden

\`\`\`json
{
  "success": false,
  "error": {
    "code": "FORBIDDEN",
    "message": "API access requires Pro plan"
  }
}
\`\`\`

**संभावित कारण:**
- Free या Starter योजना (API उपलब्ध नहीं)
- आवश्यक अनुमतियों के बिना API कुंजी

## सुरक्षा

### कुंजी रोटेशन

नियमित रूप से अपनी API कुंजी बदलें:

1. एक नई कुंजी उत्पन्न करें
2. अपने एप्लिकेशन अपडेट करें
3. सत्यापित करें कि सब कुछ काम करता है
4. पुरानी कुंजी रद्द करें

### निगरानी

अपनी कुंजी उपयोग की निगरानी करें:
- नियमित रूप से "अंतिम उपयोग" जांचें
- अप्रयुक्त कुंजियों की पहचान करें
- संदिग्ध उपयोग का पता लगाएं

### समझौता के मामले में

यदि कोई कुंजी समझौता हो जाती है:

1. FluenzR में **तुरंत रद्द करें** कुंजी
2. नई कुंजी उत्पन्न करें
3. अपने एप्लिकेशन अपडेट करें
4. दुर्भावनापूर्ण उपयोग का पता लगाने के लिए लॉग का विश्लेषण करें
5. यदि आवश्यक हो तो समर्थन से संपर्क करें
