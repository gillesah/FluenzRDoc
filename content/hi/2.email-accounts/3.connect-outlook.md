# Outlook कनेक्ट करें

FluenzR OAuth 2.0 के माध्यम से व्यक्तिगत Outlook खातों और पेशेवर Microsoft 365 खातों का समर्थन करता है।

## आवश्यकताएं

- एक Outlook.com, Hotmail, या Microsoft 365 खाता
- FluenzR में ईमेल खाते पृष्ठ तक पहुंच

## कनेक्शन चरण

### 1. कॉन्फ़िगरेशन तक पहुंचें

1. अपने FluenzR खाते में लॉग इन करें
2. साइडबार में, **Email Accounts** पर क्लिक करें
3. **+ Microsoft** बटन पर क्लिक करें

### 2. पहुंच अधिकृत करें

1. एक Microsoft विंडो खुलती है
2. अपने Microsoft खाते में लॉग इन करें
3. अनुरोधित अनुमतियों की समीक्षा करें

### 3. अनुमतियां प्रदान करें

FluenzR निम्नलिखित के लिए पहुंच का अनुरोध करता है:
- **Mail.Send**: ईमेल भेजें
- **Mail.Read**: ईमेल पढ़ें (प्रतिक्रिया पहचान)
- **User.Read**: आपकी बुनियादी प्रोफ़ाइल तक पहुंचें

पुष्टि करने के लिए **Accept** पर क्लिक करें।

### 4. सत्यापन

अधिकृतीकरण के बाद, आपका Outlook खाता FluenzR में **Connected** स्थिति के साथ दिखाई देता है।

## Outlook सीमाएं

| खाता प्रकार | दैनिक सीमा |
|--------------|-------------|
| Outlook.com / Hotmail | ~300 ईमेल/दिन |
| Microsoft 365 Business | ~10,000 ईमेल/दिन |

> **जानकारी**
पेशेवर Microsoft 365 खातों में बहुत अधिक सीमाएं हैं, जो उन्हें संभावना तलाशने के लिए आदर्श बनाती हैं।

## Microsoft 365 बनाम Outlook.com

### Outlook.com (व्यक्तिगत)

- निःशुल्क
- अधिक प्रतिबंधात्मक सीमाएं
- छोटे वॉल्यूम के लिए उपयुक्त

### Microsoft 365 (पेशेवर)

- सशुल्क (€5.60/माह से)
- कस्टम डोमेन संभव
- उच्च सीमाएं
- केंद्रीकृत प्रशासन
- बेहतर पेशेवर छवि

## Microsoft 365 के लिए कॉन्फ़िगरेशन

यदि आप कस्टम डोमेन के साथ Microsoft 365 का उपयोग करते हैं, तो सुनिश्चित करें कि:

1. आपके डोमेन के लिए **SPF** कॉन्फ़िगर किया गया है
2. **DKIM** Microsoft 365 व्यवस्थापक में सक्षम है
3. **DMARC** कॉन्फ़िगर किया गया है (अनुशंसित)

ये DNS कॉन्फ़िगरेशन आपके ईमेल डिलीवरेबिलिटी में सुधार करते हैं।

## समस्या निवारण

### "व्यवस्थापक द्वारा पहुंच अस्वीकृत"

Microsoft 365 खातों के लिए, आपके संगठन का व्यवस्थापक तीसरे पक्ष के एप्लिकेशन को ब्लॉक कर सकता है:
1. अपने IT व्यवस्थापक से संपर्क करें
2. Azure AD में FluenzR अनुमोदन का अनुरोध करें
3. या इस बीच एक व्यक्तिगत खाते का उपयोग करें

### "प्रमाणीकरण विफल"

- जांचें कि खाते में दो-कारक प्रमाणीकरण अवरुद्ध नहीं है
- Microsoft से लॉग आउट/लॉग इन करने का प्रयास करें
- एक निजी ब्राउज़र का उपयोग करें

### "अमान्य टोकन"

- FluenzR में, खाता हटाएं
- इसे शुरू से पुनः कनेक्ट करें
- यदि समस्या बनी रहती है, तो Microsoft की तरफ से पहुंच रद्द करें फिर पुनः कनेक्ट करें

## पहुंच रद्द करें

अपने Microsoft खाते से FluenzR को डिस्कनेक्ट करने के लिए:

1. [account.microsoft.com/privacy/app-access](https://account.microsoft.com/privacy/app-access) पर जाएं
2. सूची में "FluenzR" खोजें
3. **Remove these permissions** पर क्लिक करें
