# Variables y personalización

Las variables te permiten personalizar automáticamente tus emails con la información de cada contacto.

## ¿Por qué personalizar?

### Mejor tasa de apertura

Un asunto personalizado obtiene 26% más aperturas que un asunto genérico.

### Mejor engagement

Los emails personalizados tienen una tasa de respuesta 2x más alta que los emails genéricos.

### Imagen profesional

La personalización muestra que has investigado y te diriges a la persona correcta.

## Variables disponibles

### Variables estándar

| Variable | Descripción | Ejemplo |
|----------|-------------|---------|
| firstName | Nombre | John |
| lastName | Apellido | Smith |
| email | Email | john@example.com |
| company | Empresa | Acme Corp |
| jobTitle | Cargo | Director de Ventas |
| phone | Teléfono | +1 555 123 4567 |
| website | Sitio web | https://acme.com |

### Variables del sistema

| Variable | Descripción |
|----------|-------------|
| unsubscribe_link | Enlace de cancelación de suscripción |
| current_date | Fecha de hoy (formato local) |
| current_year | Año actual |

### Campos personalizados

Tus campos personalizados están disponibles como variables:

| Variable | Ejemplo |
|----------|---------|
| sector | Fintech |
| company_size | 50-100 |
| source | LinkedIn |

## Usar las variables

### En el asunto

Ejemplo: `Hola firstName, pregunta rápida`

Resultado: "Hola John, pregunta rápida"

### En el contenido

```text
Hola [firstName],

Noté que [company] se está expandiendo en el sector [sector].
Como [jobTitle], debes estar enfrentando...
```

Reemplaza los corchetes con la sintaxis de variable usando llaves dobles.

### Menú de inserción

1. En el editor de email, coloca tu cursor
2. Haz clic en **Variables** en la barra de herramientas
3. Selecciona la variable a insertar

## Valores por defecto (fallback)

Si una variable está vacía para un contacto, puedes definir un valor por defecto con la sintaxis: `variable|valor_por_defecto`

### Ejemplos

- `firstName|hola` → "John" o "hola" si el nombre está vacío
- `company|tu empresa` → "Acme Corp" o "tu empresa" si está vacío

## Mejores prácticas

### Siempre proporciona un fallback

Nunca dejes una variable sin un posible fallback para evitar emails con espacios en blanco.

### Verifica la calidad de los datos

Antes de usar una variable, asegúrate de que:
- El campo esté completado para la mayoría de contactos
- Los datos estén limpios (capitalización correcta, etc.)

### Personalización natural

Evita la sobre-personalización que parece artificial. 1-2 variables por párrafo son suficientes.

### Prueba antes de enviar

1. Usa la vista previa con diferentes contactos
2. Envía un email de prueba a ti mismo
3. Verifica casos donde faltan datos

## Personalización avanzada (Plan Pro)

### Variables condicionales

Muestra texto basado en una condición. Por ejemplo, muestra texto diferente dependiendo de si el cargo está completado o no.

### Formato

Opciones de formato disponibles:
- uppercase: Convierte a mayúsculas
- lowercase: Convierte a minúsculas
- short: Formato corto para fechas

## Ejemplo de plantilla

```text
Asunto: [firstName], pregunta sobre [company]

Hola [firstName],

Te contacto porque [company] parece encajar
perfectamente con el perfil de empresas que apoyamos.

¿Estarías disponible para un intercambio de 15 minutos?

Saludos cordiales,
[Firma]

[unsubscribe_link]
```

Reemplaza los corchetes con la sintaxis de variable usando llaves dobles.

## Resolución de problemas

### Variable mostrada tal cual

Si la variable se muestra en lugar del valor:
- Verifica la ortografía de la variable
- Verifica que el campo exista para el contacto
- Usa el menú de variables en lugar de entrada manual

### Datos incorrectos

Si "john" se muestra en lugar de "John":
- Limpia tus datos en la fuente
- Usa importación CSV con datos limpios
- Corrige manualmente los contactos importantes

### Variable vacía

Si no se muestra nada:
- Añade un fallback
- Verifica que el campo no esté vacío para este contacto
