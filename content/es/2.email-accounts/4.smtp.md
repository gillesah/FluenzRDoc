# Configuración SMTP

Para proveedores de email distintos de Gmail y Outlook, FluenzR soporta la conexión vía SMTP.

## ¿Cuándo usar SMTP?

- Servidor de email auto-alojado
- Proveedor de email francés (OVH, Gandi, etc.)
- Servicio de email profesional (Zoho, ProtonMail Bridge, etc.)
- Necesidades específicas de configuración

> **⚠️ Advertencia**
La conexión SMTP almacena tus credenciales de forma encriptada (AES-256). Para máxima seguridad, prefiere OAuth (Gmail/Outlook) cuando sea posible.

## Información requerida

Para configurar SMTP, necesitarás:

| Parámetro | Descripción | Ejemplo |
|-----------|-------------|---------|
| **Servidor SMTP** | Dirección del servidor saliente | smtp.example.com |
| **Puerto** | Puerto de conexión | 587 (TLS) o 465 (SSL) |
| **Nombre de usuario** | Generalmente tu email | user@example.com |
| **Contraseña** | Contraseña o app password | ********** |
| **Encriptación** | Tipo de seguridad | TLS (recomendado) |

## Configuración paso a paso

### 1. Acceder a la configuración

1. Ve a **Cuentas de Email**
2. Haz clic en **+ SMTP**

### 2. Completar la información

1. **Email**: Tu dirección de email completa
2. **Nombre visible**: El nombre que aparecerá como remitente
3. **Servidor SMTP**: La dirección de tu servidor
4. **Puerto**: Generalmente 587 (TLS) o 465 (SSL)
5. **Nombre de usuario**: A menudo idéntico al email
6. **Contraseña**: Tu contraseña (o app password)
7. **Encriptación**: Selecciona TLS o SSL

### 3. Probar la conexión

Haz clic en **Probar** para verificar que la configuración funciona. Se enviará un email de prueba a tu propia dirección.

### 4. Configurar IMAP (opcional)

Para la detección de respuestas, configura también IMAP:

| Parámetro | Descripción | Ejemplo |
|-----------|-------------|---------|
| **Servidor IMAP** | Dirección del servidor entrante | imap.example.com |
| **Puerto** | Puerto de conexión | 993 (SSL) |

## Configuraciones comunes

### OVH Mail

```
SMTP: ssl0.ovh.net
Puerto: 587 (TLS)
IMAP: ssl0.ovh.net
Puerto IMAP: 993 (SSL)
```

### Gandi

```
SMTP: mail.gandi.net
Puerto: 587 (TLS)
IMAP: mail.gandi.net
Puerto IMAP: 993 (SSL)
```

### Zoho Mail

```
SMTP: smtp.zoho.eu (o smtp.zoho.com)
Puerto: 587 (TLS)
IMAP: imap.zoho.eu
Puerto IMAP: 993 (SSL)
```

> **ℹ️ Información**
Para Zoho, necesitarás generar un "App Password" en la configuración de seguridad de tu cuenta.

### Infomaniak

```
SMTP: mail.infomaniak.com
Puerto: 587 (TLS)
IMAP: mail.infomaniak.com
Puerto IMAP: 993 (SSL)
```

### ProtonMail (vía Bridge)

```
SMTP: 127.0.0.1
Puerto: 1025
IMAP: 127.0.0.1
Puerto IMAP: 1143
```

> **⚠️ Advertencia**
ProtonMail requiere la aplicación Bridge instalada y activa en tu máquina.

## App Passwords

Algunos proveedores requieren una "contraseña de aplicación" en lugar de tu contraseña principal:

### ¿Cuándo usar un App Password?

- Autenticación de dos factores (2FA) activada
- Requisito de seguridad del proveedor
- Política de la organización

### ¿Cómo generarlo?

1. Inicia sesión en la interfaz web de tu proveedor
2. Ve a Configuración > Seguridad
3. Busca "Contraseñas de aplicación" o "App Passwords"
4. Crea una nueva contraseña para "FluenzR"
5. Copia la contraseña generada en FluenzR

## Resolución de problemas

### "Conexión rechazada"

- Verifica el puerto y el servidor
- Asegúrate de que tu firewall no esté bloqueando
- Prueba con otro puerto (465 en lugar de 587)

### "Autenticación fallida"

- Verifica el nombre de usuario (a menudo el email completo)
- Usa un App Password si 2FA está activado
- Verifica las mayúsculas/minúsculas de la contraseña

### "Certificado inválido"

- Asegúrate de usar el tipo correcto de encriptación
- Contacta a tu proveedor si el certificado ha expirado

### "Límite de envío alcanzado"

Los servidores SMTP a menudo tienen límites:
- Verifica los límites de tu proveedor
- Espacía tus envíos
- Contacta a tu host para aumentar las cuotas

## Seguridad

### Almacenamiento de credenciales

- Las credenciales SMTP están encriptadas en AES-256
- Nunca se almacenan en texto plano en la base de datos
- Accesibles solo por el sistema de envío

### Recomendaciones

1. Usa un App Password dedicado
2. Activa las alertas de inicio de sesión con tu proveedor
3. Cambia la contraseña regularmente
4. Prefiere OAuth cuando sea posible
