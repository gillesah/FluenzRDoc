# Conectar Outlook

FluenzR soporta cuentas personales de Outlook y cuentas profesionales de Microsoft 365 vía OAuth 2.0.

## Requisitos previos

- Una cuenta Outlook.com, Hotmail o Microsoft 365
- Acceso a la página de cuentas de email en FluenzR

## Pasos de conexión

### 1. Acceder a la configuración

1. Inicia sesión en tu cuenta de FluenzR
2. En el menú lateral, haz clic en **Cuentas de Email**
3. Haz clic en el botón **+ Microsoft**

### 2. Autorizar el acceso

1. Se abre una ventana de Microsoft
2. Inicia sesión en tu cuenta de Microsoft
3. Revisa los permisos solicitados

### 3. Conceder permisos

FluenzR solicita acceso para:
- **Mail.Send**: Enviar emails
- **Mail.Read**: Leer emails (detección de respuestas)
- **User.Read**: Acceder a tu perfil básico

Haz clic en **Aceptar** para confirmar.

### 4. Verificación

Tras la autorización, tu cuenta de Outlook aparece en FluenzR con el estado **Conectada**.

## Límites de Outlook

| Tipo de cuenta | Límite diario |
|----------------|---------------|
| Outlook.com / Hotmail | ~300 emails/día |
| Microsoft 365 Business | ~10.000 emails/día |

> **ℹ️ Información**
Las cuentas profesionales de Microsoft 365 tienen límites mucho más altos, lo que las hace ideales para prospección.

## Microsoft 365 vs Outlook.com

### Outlook.com (personal)
- Gratuito
- Límites más restrictivos
- Adecuado para volúmenes pequeños

### Microsoft 365 (profesional)
- De pago (desde 5,60€/mes)
- Dominio personalizado posible
- Límites altos
- Administración centralizada
- Mejor imagen profesional

## Configuración para Microsoft 365

Si usas Microsoft 365 con un dominio personalizado, asegúrate de que:

1. **SPF** está configurado para tu dominio
2. **DKIM** está activado en el admin de Microsoft 365
3. **DMARC** está configurado (recomendado)

Estas configuraciones DNS mejoran la deliverability de tus emails.

## Resolución de problemas

### "Acceso denegado por el administrador"
Para cuentas de Microsoft 365, el administrador de tu organización puede bloquear aplicaciones de terceros:
1. Contacta a tu administrador de IT
2. Solicita la aprobación de FluenzR en Azure AD
3. O usa una cuenta personal mientras tanto

### "Fallo de autenticación"
- Verifica que la cuenta no tenga autenticación de dos factores bloqueante
- Intenta cerrar sesión/iniciar sesión en Microsoft
- Usa un navegador privado

### "Token inválido"
- En FluenzR, elimina la cuenta
- Reconéctala desde el principio
- Si el problema persiste, revoca el acceso en Microsoft y reconecta

## Revocar acceso

Para desconectar FluenzR de tu cuenta de Microsoft:

1. Ve a [account.microsoft.com/privacy/app-access](https://account.microsoft.com/privacy/app-access)
2. Encuentra "FluenzR" en la lista
3. Haz clic en **Eliminar estos permisos**
