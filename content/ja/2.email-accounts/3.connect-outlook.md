# Connecter Outlook

FluenzR supporte les comptes Outlook personnels et Microsoft 365 professionnels via OAuth 2.0.

## Prérequis

- Un compte Outlook.com, Hotmail, ou Microsoft 365
- Accès à la page des comptes email dans FluenzR

## Étapes de connexion

### 1. Accéder à la configuration

1. Connectez-vous à votre compte FluenzR
2. Dans le menu latéral, cliquez sur **Comptes Email**
3. Cliquez sur le bouton **+ Microsoft**

### 2. Autoriser l'accès

1. Une fenêtre Microsoft s'ouvre
2. Connectez-vous à votre compte Microsoft
3. Vérifiez les permissions demandées

### 3. Accorder les permissions

FluenzR demande l'accès pour :
- **Mail.Send** : Envoyer des emails
- **Mail.Read** : Lire les emails (détection des réponses)
- **User.Read** : Accéder à votre profil de base

Cliquez sur **Accepter** pour confirmer.

### 4. Vérification

Après l'autorisation, votre compte Outlook apparaît dans FluenzR avec le statut **Connecté**.

## Limites Outlook

| Type de compte | Limite quotidienne |
|----------------|-------------------|
| Outlook.com / Hotmail | ~300 emails/jour |
| Microsoft 365 Business | ~10 000 emails/jour |

> **ℹ️ Info**
Les comptes Microsoft 365 professionnels ont des limites beaucoup plus élevées, ce qui les rend idéaux pour la prospection.

## Microsoft 365 vs Outlook.com

### Outlook.com (personnel)
- Gratuit
- Limites plus restrictives
- Convient pour les petits volumes

### Microsoft 365 (professionnel)
- Payant (à partir de 5,60€/mois)
- Domaine personnalisé possible
- Limites élevées
- Administration centralisée
- Meilleure image professionnelle

## Configuration pour Microsoft 365

Si vous utilisez Microsoft 365 avec un domaine personnalisé, assurez-vous que :

1. **SPF** est configuré pour votre domaine
2. **DKIM** est activé dans l'admin Microsoft 365
3. **DMARC** est configuré (recommandé)

Ces configurations DNS améliorent la délivrabilité de vos emails.

## Résolution de problèmes

### "Accès refusé par l'administrateur"
Pour les comptes Microsoft 365, l'administrateur de votre organisation peut bloquer les applications tierces :
1. Contactez votre administrateur IT
2. Demandez l'approbation de FluenzR dans Azure AD
3. Ou utilisez un compte personnel en attendant

### "Échec de l'authentification"
- Vérifiez que le compte n'a pas d'authentification à deux facteurs bloquante
- Essayez de vous déconnecter/reconnecter à Microsoft
- Utilisez un navigateur privé

### "Token invalide"
- Dans FluenzR, supprimez le compte
- Reconnectez-le depuis le début
- Si le problème persiste, révoquez l'accès côté Microsoft puis reconnectez

## Révocation d'accès

Pour déconnecter FluenzR de votre compte Microsoft :

1. Allez sur [account.microsoft.com/privacy/app-access](https://account.microsoft.com/privacy/app-access)
2. Trouvez "FluenzR" dans la liste
3. Cliquez sur **Supprimer ces autorisations**
