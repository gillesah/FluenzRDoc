# Outlookの接続

FluenzRは、OAuth 2.0を介して個人のOutlookアカウントとMicrosoft 365ビジネスアカウントをサポートしています。

## 前提条件

- Outlook.com、Hotmail、またはMicrosoft 365アカウント
- FluenzRのメールアカウントページへのアクセス

## 接続手順

### 1. 設定にアクセス

1. FluenzRアカウントにログインします
2. サイドバーで**メールアカウント**をクリックします
3. **+ Microsoft**ボタンをクリックします

### 2. アクセスを承認

1. Microsoftウィンドウが開きます
2. Microsoftアカウントにログインします
3. 要求された権限を確認します

### 3. 権限を付与

FluenzRは以下のアクセスを要求します:
- **Mail.Send**: メールの送信
- **Mail.Read**: メールの読み取り(返信の検出)
- **User.Read**: 基本プロフィールへのアクセス

**承認**をクリックして確認します。

### 4. 確認

承認後、OutlookアカウントがFluenzRに**接続済み**ステータスで表示されます。

## Outlookの制限

| アカウントタイプ | 1日の制限 |
|--------------|-------------|
| Outlook.com / Hotmail | ~300通/日 |
| Microsoft 365 Business | ~10,000通/日 |

> **情報**
Microsoft 365ビジネスアカウントははるかに高い制限があり、営業活動に理想的です。

## Microsoft 365 vs Outlook.com

### Outlook.com(個人)
- 無料
- より制限的な制限
- 小規模なボリュームに適している

### Microsoft 365(ビジネス)
- 有料(€5.60/月から)
- カスタムドメインが可能
- 高い制限
- 一元管理
- より良いプロフェッショナルイメージ

## Microsoft 365の設定

カスタムドメインでMicrosoft 365を使用している場合は、以下を確認してください:

1. ドメインに**SPF**が設定されている
2. Microsoft 365管理画面で**DKIM**が有効になっている
3. **DMARC**が設定されている(推奨)

これらのDNS設定により、メールの配信性が向上します。

## トラブルシューティング

### 「管理者によってアクセスが拒否されました」
Microsoft 365アカウントの場合、組織の管理者がサードパーティアプリケーションをブロックしている可能性があります:
1. IT管理者に連絡してください
2. Azure ADでFluenzRの承認を依頼してください
3. または、当面は個人アカウントを使用してください

### 「認証に失敗しました」
- アカウントに二要素認証のブロックがないか確認してください
- Microsoftからログアウト/ログインしてみてください
- プライベートブラウザを使用してください

### 「トークンが無効です」
- FluenzRでアカウントを削除します
- 最初から再接続します
- 問題が続く場合は、Microsoft側でアクセスを取り消してから再接続してください

## アクセスの取り消し

MicrosoftアカウントからFluenzRを切断するには:

1. [account.microsoft.com/privacy/app-access](https://account.microsoft.com/privacy/app-access)にアクセスします
2. リストから「FluenzR」を見つけます
3. **これらの権限を削除**をクリックします

## Gmail vs Outlook の比較

| 基準 | Gmail | Outlook |
|----------|-------|---------|
| 個人の制限 | 500通/日 | 300通/日 |
| ビジネスの制限 | 2000通/日 | 10000通/日 |
| 設定 | 簡単 | 簡単 |
| 配信性 | 優秀 | 非常に良い |
| ビジネス価格 | €5.75/月 | €5.60/月 |

> **成功**
最大の柔軟性を得るには、StarterまたはProプランでGmailとOutlookアカウントを組み合わせてください。
