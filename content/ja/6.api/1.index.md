# REST API

FluenzR APIを使用すると、プラットフォームをツールと統合し、ワークフローを自動化できます。

> **ℹ️ 情報**
REST APIは**Pro**プランでのみ利用可能です。

## 概要

FluenzR APIはREST APIで、以下のことが可能です：

- 連絡先をプログラムで管理
- キャンペーンの作成と制御
- 統計情報の取得
- イベントのWebhook受信

## ベースURL

```
https://api.fluenzr.co/v1
```

## レスポンス形式

すべてのレスポンスはJSON形式です：

```json
{
  "success": true,
  "data": { ... },
  "meta": {
    "page": 1,
    "perPage": 20,
    "total": 150
  }
}
```

### エラーの場合

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Email is required",
    "details": { ... }
  }
}
```

## HTTPコード

| コード | 意味 |
|------|---------------|
| 200 | 成功 |
| 201 | リソース作成成功 |
| 400 | 無効なリクエスト |
| 401 | 未認証 |
| 403 | 権限なし |
| 404 | リソースが見つからない |
| 429 | レート制限超過 |
| 500 | サーバーエラー |

## レート制限

| プラン | 制限 |
|------|--------|
| Pro | 1000リクエスト/分 |

レスポンスヘッダーで使用状況を確認できます：

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 950
X-RateLimit-Reset: 1640000000
```

## ページネーション

リソース一覧を返すエンドポイントはページネーションをサポートしています：

```
GET /contacts?page=2&perPage=50
```

| パラメータ | 説明 | デフォルト | 最大 |
|-----------|-------------|--------|-----|
| `page` | ページ番号 | 1 | - |
| `perPage` | ページあたりの件数 | 20 | 100 |

## フィルタリング

クエリパラメータで結果をフィルタリング：

```
GET /contacts?email=example.com&group=123
```

## ソート

結果のソート：

```
GET /contacts?sort=createdAt&order=desc
```

## 利用可能なエンドポイント

### 連絡先

| メソッド | エンドポイント | 説明 |
|---------|----------|-------------|
| GET | /contacts | 連絡先一覧 |
| GET | /contacts/:id | 連絡先詳細 |
| POST | /contacts | 連絡先作成 |
| PUT | /contacts/:id | 連絡先更新 |
| DELETE | /contacts/:id | 連絡先削除 |

### グループ

| メソッド | エンドポイント | 説明 |
|---------|----------|-------------|
| GET | /groups | グループ一覧 |
| GET | /groups/:id | グループ詳細 |
| POST | /groups | グループ作成 |
| PUT | /groups/:id | グループ更新 |
| DELETE | /groups/:id | グループ削除 |
| POST | /groups/:id/contacts | 連絡先追加 |

### キャンペーン

| メソッド | エンドポイント | 説明 |
|---------|----------|-------------|
| GET | /campaigns | キャンペーン一覧 |
| GET | /campaigns/:id | キャンペーン詳細 |
| POST | /campaigns | キャンペーン作成 |
| PUT | /campaigns/:id | キャンペーン更新 |
| POST | /campaigns/:id/start | 開始 |
| POST | /campaigns/:id/pause | 一時停止 |
| POST | /campaigns/:id/contacts | 連絡先追加 |
| GET | /campaigns/:id/stats | 統計 |

### Webhooks

| メソッド | エンドポイント | 説明 |
|---------|----------|-------------|
| GET | /webhooks | Webhook一覧 |
| POST | /webhooks | Webhook作成 |
| DELETE | /webhooks/:id | 削除 |

## SDKとライブラリ

### JavaScript/TypeScript

```bash
npm install @fluenzr/sdk
```

```javascript
import { FluenzrClient } from '@fluenzr/sdk';

const client = new FluenzrClient({
  apiKey: 'your-api-key'
});

const contacts = await client.contacts.list();
```

### Python

```bash
pip install fluenzr
```

```python
from fluenzr import FluenzrClient

client = FluenzrClient(api_key='your-api-key')
contacts = client.contacts.list()
```

## 使用例

### CRM同期

```javascript
// CRMから新しい連絡先を取得
const crmContacts = await myCRM.getNewContacts();

// FluenzRに作成
for (const contact of crmContacts) {
  await fluenzr.contacts.create({
    email: contact.email,
    firstName: contact.first_name,
    lastName: contact.last_name,
    company: contact.company
  });
}
```

### キャンペーン自動化

```javascript
// API経由でキャンペーンを作成
const campaign = await fluenzr.campaigns.create({
  name: 'Automated Q1 2025',
  emailAccountId: 'acc_123',
  // ...
});

// 連絡先を追加
await fluenzr.campaigns.addContacts(campaign.id, {
  groupId: 'grp_456'
});

// 開始
await fluenzr.campaigns.start(campaign.id);
```

### 統計エクスポート

```javascript
// すべてのキャンペーンの統計を取得
const campaigns = await fluenzr.campaigns.list();

for (const campaign of campaigns) {
  const stats = await fluenzr.campaigns.getStats(campaign.id);
  console.log(`${campaign.name}: ${stats.openRate}% opens, ${stats.replyRate}% replies`);
}
```
