# ワークフローエディター

FluenzRのワークフローエディターでは、直感的なドラッグ＆ドロップインターフェースでメールシーケンスを視覚的に作成できます。

## インターフェース

### 主要エリア

| エリア | 説明 |
|------|-------------|
| **ツールバー** | ドラッグ可能なノード |
| **キャンバス** | ワークフローの作業エリア |
| **設定パネル** | 選択したノードの設定 |

### ナビゲーション

- **ズーム**: マウスホイールまたは+/-ボタン
- **移動**: キャンバス上で左クリックしながらドラッグ
- **選択**: ノードをクリック

## ノードの種類

### 開始

開始ノードは自動的に配置されます。シーケンスのエントリーポイントです。

- 削除不可
- 複製不可
- ワークフローごとに1つのみ

### メール

連絡先にパーソナライズされたメールを送信します。

**設定:**

| フィールド | 説明 |
|-------|-------------|
| **名前** | ノードの識別名（内部用） |
| **件名** | メールの件名 |
| **本文** | メールの本文 |
| **トラッキング** | 開封とクリック |

**機能:**
- リッチテキストエディター
- 動的変数
- リアルタイムプレビュー

### 待機

2つのアクション間の一時停止。

**設定:**

| パラメータ | オプション |
|-----------|---------|
| **期間** | 1-30日または1-24時間 |
| **種類** | カレンダー日または営業日 |

**例:**
- "3日": 72時間待機
- "3営業日": 3営業日（月-金）待機、カレンダー上は5日になる可能性あり

### 条件（Proプラン）

条件に基づいて分岐を作成。

**利用可能な条件:**

| 条件 | 説明 |
|---------|-------------|
| メール開封 | 連絡先が最後のメールを開封した |
| リンククリック | 連絡先がリンクをクリックした |
| 前のメール | 前のメールの状態 |
| 連絡先フィールド | 連絡先のフィールド値 |

**構造:**
```
[条件]
   ├─ 真の場合 → [メールA]
   └─ 偽の場合 → [メールB]
```

### 終了

この分岐のシーケンス終了をマーク。

- 連絡先の経路を適切に終了
- 複数の終了ノードを持つことが可能（異なる分岐用）

## 接続を作成

### 2つのノードを接続

1. ノードの出力ポイント（下部）をクリック
2. 次のノードの入力ポイント（上部）にドラッグ
3. リリースして接続を作成

### 接続を削除

1. 接続（線）をクリック
2. **Delete**キーを押すかゴミ箱アイコンをクリック

## メールノードを編集

### エディターを開く

メールノードをダブルクリックして編集パネルを開きます。

### メールの件名

```
{{firstName}}さん、簡単な質問
```

ヒント：
- 最大50文字
- 変数を使ってパーソナライズ
- 大文字や感嘆符を避ける

### メールの本文

エディターの機能：

| 機能 | 説明 |
|----------|-------------|
| **太字/イタリック** | 基本的な書式 |
| **リスト** | 箇条書きと番号付き |
| **リンク** | クリック可能なハイパーリンク |
| **変数** | 変数挿入メニュー |

### 詳細オプション

| オプション | 説明 | デフォルト |
|--------|-------------|--------|
| 開封トラッキング | 開封者を追跡 | 有効 |
| クリックトラッキング | クリックを追跡 | 有効 |

## ワークフローのベストプラクティス

### 推奨構造

効果的なシーケンス：

```
[開始]
    ↓
[メール1] - イントロダクション
    ↓
[3日待機]
    ↓
[メール2] - 付加価値
    ↓
[4日待機]
    ↓
[メール3] - 最後のチャンス
    ↓
[終了]
```

### 最適なメール数

| キャンペーンタイプ | 推奨メール数 |
|------------------|-------------------|
| コールドプロスペクティング | 3-4通 |
| ナーチャリング | 5-7通 |
| イベント | 2-3通 |
| 顧客フォローアップ | 2-3通 |

### メール間の間隔

| コンテキスト | 推奨間隔 |
|----------|------------------|
| メール1と2の間 | 3-4日 |
| メール2と3の間 | 4-5日 |
| メール3と4の間 | 5-7日 |

> **注意**
間隔が短すぎると見込み客を困らせる可能性があります。長すぎると勢いを失います。

## 条件分岐ワークフロー（Proプラン）

### 例: 開封に基づくフォローアップ

```
[メール1]
    ↓
[3日待機]
    ↓
[条件: メール1を開封？]
   ├─ はい → [メール2: パーソナライズされたフォローアップ]
   └─ いいえ → [メール2: 新しいアプローチ]
```

### 例: クリックに基づく絞り込み

```
[2つのリンクを含むメール]
    ↓
[2日待機]
    ↓
[条件: リンクAをクリック？]
   ├─ はい → [ソリューションA向けメール]
   └─ いいえ → [条件: リンクBをクリック？]
              ├─ はい → [ソリューションB向けメール]
              └─ いいえ → [一般的なメール]
```

## キーボードショートカット

| ショートカット | アクション |
|-----------|--------|
| `Ctrl + Z` | 元に戻す |
| `Ctrl + Y` | やり直し |
| `Delete` | 選択したノードを削除 |
| `Ctrl + D` | ノードを複製 |
| `Ctrl + S` | 保存 |

## トラブルシューティング

### "ワークフローが無効です"

確認事項：
- すべてのノードが接続されている
- ループがない
- 各分岐が終了ノードで終わっている

### "コンテンツのないメール"

各メールノードに必要なもの：
- 空でない件名
- 空でない本文

### 未接続のノード

グレーアウトまたは赤いインジケーターのあるノードは接続されていません。ワークフローに接続するか削除してください。
