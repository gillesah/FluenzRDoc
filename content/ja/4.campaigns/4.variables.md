# Variables et personnalisation

Les variables vous permettent de personnaliser automatiquement vos emails avec les informations de chaque contact.

## Pourquoi personnaliser ?

### Meilleur taux d'ouverture

Un objet personnalisé obtient 26% d'ouvertures en plus qu'un objet générique.

### Meilleur engagement

Les emails personnalisés ont un taux de réponse 2x plus élevé que les emails génériques.

### Image professionnelle

La personnalisation montre que vous avez fait des recherches et que vous vous adressez à la bonne personne.

## Variables disponibles

### Variables standard

| Variable | Description | Exemple |
|----------|-------------|---------|
| firstName | Prénom | Jean |
| lastName | Nom | Dupont |
| email | Email | jean@example.com |
| company | Entreprise | Acme Corp |
| jobTitle | Poste | Directeur Commercial |
| phone | Téléphone | +33 6 12 34 56 78 |
| website | Site web | https://acme.com |

### Variables système

| Variable | Description |
|----------|-------------|
| unsubscribe_link | Lien de désinscription |
| current_date | Date du jour (format local) |
| current_year | Année en cours |

### Champs personnalisés

Vos champs personnalisés sont disponibles comme variables :

| Variable | Exemple |
|----------|---------|
| secteur | Fintech |
| taille_entreprise | 50-100 |
| source | LinkedIn |

## Utiliser les variables

### Dans l'objet

Exemple : `Bonjour firstName, question rapide`

Résultat : "Bonjour Jean, question rapide"

### Dans le contenu

```text
Bonjour [firstName],

J'ai vu que [company] se développe dans le secteur [secteur].
En tant que [jobTitle], vous devez faire face à...
```

Remplacez les crochets par la syntaxe de variable avec double accolades.

### Menu d'insertion

1. Dans l'éditeur d'email, placez votre curseur
2. Cliquez sur **Variables** dans la barre d'outils
3. Sélectionnez la variable à insérer

## Valeurs par défaut (fallback)

Si une variable est vide pour un contact, vous pouvez définir une valeur par défaut avec la syntaxe : `variable|valeur_par_defaut`

### Exemples

- `firstName|there` → "Jean" ou "there" si prénom vide
- `company|votre entreprise` → "Acme Corp" ou "votre entreprise" si vide

## Bonnes pratiques

### Toujours prévoir un fallback

Ne laissez jamais une variable sans fallback possible pour éviter les emails avec des blancs.

### Vérifier la qualité des données

Avant d'utiliser une variable, assurez-vous que :
- Le champ est renseigné pour la majorité des contacts
- Les données sont propres (majuscules correctes, etc.)

### Personnalisation naturelle

Évitez la sur-personnalisation qui semble artificielle. 1-2 variables par paragraphe suffisent.

### Tester avant d'envoyer

1. Utilisez l'aperçu avec différents contacts
2. Envoyez un email test à vous-même
3. Vérifiez les cas où des données manquent

## Personnalisation avancée (Plan Pro)

### Variables conditionnelles

Affichez du texte selon une condition. Par exemple, afficher un texte différent selon que le poste est renseigné ou non.

### Formatage

Options de formatage disponibles :
- uppercase : Met en majuscules
- lowercase : Met en minuscules
- short : Format court pour les dates

## Exemple de template

```text
Objet : [firstName], question sur [company]

Bonjour [firstName],

Je me permets de vous contacter car [company] semble
correspondre parfaitement au profil des entreprises que
nous accompagnons.

Seriez-vous disponible pour un échange de 15 minutes ?

Cordialement,
[Signature]

[unsubscribe_link]
```

Remplacez les crochets par la syntaxe de variable avec double accolades.

## Résolution de problèmes

### Variable affichée telle quelle

Si la variable s'affiche au lieu de la valeur :
- Vérifiez l'orthographe de la variable
- Vérifiez que le champ existe pour le contact
- Utilisez le menu de variables plutôt que la saisie manuelle

### Données incorrectes

Si "jean" s'affiche au lieu de "Jean" :
- Nettoyez vos données à la source
- Utilisez l'import CSV avec des données propres
- Corrigez manuellement les contacts importants

### Variable vide

Si rien ne s'affiche :
- Ajoutez un fallback
- Vérifiez que le champ n'est pas vide pour ce contact
