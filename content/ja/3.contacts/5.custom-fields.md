# カスタムフィールド

カスタムフィールドを使用すると、標準フィールド以外の特定の情報を連絡先に追加できます。

## なぜカスタムフィールドを使うのか？

### 高度なパーソナライゼーション

メールでカスタム変数を使用：
- `{{secteur}}` → "Fintech"
- `{{companySize}}` → "50-100名"
- `{{leadSource}}` → "LinkedIn Sales Navigator"

### セグメンテーション

ビジネス基準に従って連絡先をフィルタリングおよびセグメント化。

### レポーティング

カスタムディメンションに基づいてパフォーマンスを分析。

## カスタムフィールドを作成

### 1. 設定にアクセス

1. **設定**に移動
2. **カスタムフィールド**をクリック
3. **+ 新規フィールド**をクリック

### 2. フィールドを設定

| パラメータ | 説明 | 例 |
|-----------|-------------|---------|
| **名前** | 技術識別子（スペースなし） | `business_sector` |
| **ラベル** | 表示名 | "業種" |
| **タイプ** | データタイプ | テキスト、数値、日付、リスト |
| **必須** | 必須フィールドか？ | いいえ |
| **デフォルト値** | 未入力時の値 | - |

### 3. 利用可能なフィールドタイプ

| タイプ | 説明 | 使用例 |
|------|-------------|----------------------|
| **テキスト** | 自由テキスト文字列 | メモ、コメント |
| **数値** | 数値 | 売上、従業員数 |
| **日付** | 日付（年/月/日） | 最初の連絡日 |
| **ドロップダウン** | 定義されたオプションから選択 | 業種、ソース |
| **チェックボックス** | はい/いいえ | GDPR同意 |
| **URL** | ウェブリンク | LinkedInプロフィール |

## 便利なフィールド例

### B2Bプロスペクティング向け

| フィールド | タイプ | 変数 |
|-------|------|----------|
| 業種 | リスト | `{{sector}}` |
| 会社規模 | リスト | `{{companySize}}` |
| 年間売上 | 数値 | `{{revenue}}` |
| リードソース | リスト | `{{source}}` |
| LinkedInプロフィール | URL | `{{linkedinUrl}}` |

### 営業フォローアップ向け

| フィールド | タイプ | 変数 |
|-------|------|----------|
| 最初の連絡日 | 日付 | `{{contactDate}}` |
| 絞り込みスコア | 数値 | `{{score}}` |
| 興味のある製品 | リスト | `{{interestedProduct}}` |
| 予算見込み | 数値 | `{{budget}}` |

### コンプライアンス向け

| フィールド | タイプ | 変数 |
|-------|------|----------|
| GDPR同意 | チェックボックス | `{{gdprOk}}` |
| 同意日 | 日付 | `{{gdprDate}}` |
| 収集元 | テキスト | `{{collectionSource}}` |

## カスタムフィールドの使用

### フォームで

カスタムフィールドは自動的に以下に表示されます：
- 連絡先作成フォーム
- 連絡先編集フォーム
- CSVインポート（列マッピング）

### メールで

テンプレートで`{{フィールド名}}`構文を使用：

```
{{firstName}}様

{{sector}}のエキスパートとして、ご存知かと思いますが...

{{companySize}}名のチームをお持ちでしたら、
きっと以下が必要かと...
```

### フィルターで

1. **連絡先**に移動
2. **フィルター**をクリック
3. カスタムフィールドを選択
4. 条件を定義（等しい、含むなど）

## カスタムフィールドでのインポート

### CSV形式

列にフィールドの技術名を使用：

```csv
email,firstName,lastName,business_sector,company_size
taro@example.com,太郎,山田,Fintech,50-100
hanako@example.com,花子,鈴木,SaaS,10-50
```

### マッピング

インポート時、FluenzRはカスタムフィールドに対応する列を自動的に検出します。

## カスタムフィールドを編集

> **注意**
フィールドを変更すると、既存のデータに影響を与える可能性があります。慎重に行ってください。

### 変更可能なもの

- ラベル（表示名）
- デフォルト値
- ドロップダウンのオプション（追加のみ）

### 変更不可能なもの

- 技術名（識別子）
- フィールドタイプ

タイプを変更するには、新しいフィールドを作成してデータを移行してください。

## カスタムフィールドを削除

1. **設定** > **カスタムフィールド**に移動
2. 削除アイコンをクリック
3. 確認

> **危険**
削除は取り消せません。このフィールドのすべての値がすべての連絡先で失われます。

## ベストプラクティス

### 命名

| ✅ 推奨 | ❌ 避ける |
|---------------|-------------|
| `business_sector` | `業種` |
| `company_size` | `size` |
| `first_contact_date` | `date1` |

### 整理

- フィールド数を制限（最大10-15）
- 自由テキストよりリストを優先
- 各フィールドの使用法を文書化
- 未使用のフィールドを定期的にレビュー

### デフォルト値

- 適切なデフォルト値を使用
- リストでは空よりも「未入力」
- 一貫した値でフィルタリングを容易に
