# 連絡先グループ

グループを使用すると、連絡先を整理し、キャンペーンを正確にターゲットできます。FluenzRは**静的**と**動的**の2種類のグループを提供しています。

## なぜグループを使うのか？

### 整理

- ソース別に連絡先を分類（LinkedIn、展示会、ウェブサイト）
- 業種や会社規模で分ける
- 顧客と見込み客を区別

### ターゲティング

- セグメント別に特定のキャンペーンを作成
- グループに応じてメッセージをパーソナライズ
- セグメント別にパフォーマンスを測定

### 管理

- セグメントを簡単にエクスポート
- 一括アクションを適用
- 各グループの推移を追跡

## グループの種類

FluenzRは異なるニーズに対応する2種類のグループを提供：

### 静的グループ

静的グループは連絡先を管理する従来の方法です：

| 機能 | 説明 |
|---------|-------------|
| **手動追加** | 連絡先は手動またはインポートで追加 |
| **永続性** | 連絡先は明示的に削除されるまでグループに残る |
| **完全制御** | グループのメンバーを正確に決定 |
| **使用例** | 固定リスト、CSVインポート、ワンショットセグメント |

### 動的グループ

動的グループはキャンペーンへの自動登録を可能にします：

| 機能 | 説明 |
|---------|-------------|
| **自動登録** | 新しい連絡先がアクティブなキャンペーンに自動的に登録 |
| **リアルタイム** | 連絡先がグループに追加されるとすぐに、リンクされたキャンペーンに参加 |
| **スケーラブル** | 連続フロー（API、フォーム、エンリッチメント）に最適 |
| **使用例** | インバウンドリード、自動ワークフロー、継続的キャンペーン |

> **ヒント**
> 新しい連絡先が定期的に到着する継続的なナーチャリングキャンペーンには動的グループを使用してください。

## グループを作成

### 1. 作成にアクセス

1. **連絡先**に移動
2. サイドバーで**グループ**をクリック
3. **+ 新規グループ**をクリック

### 2. グループを設定

| フィールド | 説明 | 例 |
|-------|-------------|---------|
| **名前** | グループ識別子 | "LinkedInリードQ4" |
| **説明** | 説明メモ（任意） | "2024年Q4にLinkedIn経由で収集した連絡先" |
| **タイプ** | 静的または動的 | 動的 |
| **色** | 識別色 | 青 |

### 3. タイプを選択

#### 静的グループ
**静的**を選択する場合：
- 固定の連絡先リストをインポートする
- 完全な手動制御が必要
- リストが頻繁に変更されない

#### 動的グループ
**動的**を選択する場合：
- 新しい連絡先が定期的に到着（API、フォーム）
- キャンペーンに自動的に登録させたい
- 継続的なナーチャリングキャンペーンがある

### 4. 保存

**作成**をクリックして完了。

## 動的グループの仕組み

### キャンペーンへの自動登録

連絡先が**動的**グループに追加されると：

1. FluenzRがこのグループにリンクされたアクティブなキャンペーンがあるか確認
2. ある場合、連絡先はこれらのキャンペーンに自動的に登録
3. キャンペーンワークフローがこの連絡先に対してすぐに開始

```
新しい連絡先 → 動的グループ → アクティブなキャンペーン → ワークフロー開始
```

### 必要条件

自動登録が機能するには、連絡先が以下を満たす必要があります：

- ✅ 有効なメールアドレスを持っている
- ✅ **動的**タイプのグループに所属
- ✅ **アクティブ**ステータスである
- ✅ まだキャンペーンに登録されていない

### ワークフロー例

1. 動的グループ「インバウンドリード」を作成
2. ナーチャリングキャンペーンを作成し、このグループに関連付け
3. キャンペーンを開始
4. グループに追加された新しいリード（API、インポート、フォーム経由）：
   - キャンペーンに自動的に登録
   - ワークフローの最初のメールを受信
   - シーケンスを進行

> **注意**
> 動的グループは複数のアクティブなキャンペーンにリンクできます。各新しい連絡先は、グループにリンクされた**すべての**アクティブなキャンペーンに登録されます。

## グループに連絡先を追加

### 方法1: 作成時

連絡先の作成またはインポート時に、宛先グループを選択。

### 方法2: 連絡先リストから

1. 連絡先を選択（チェックボックス）
2. **アクション** > **グループに追加**をクリック
3. グループを選択
4. 確認

### 方法3: 連絡先カードから

1. 連絡先のカードを開く
2. 「グループ」セクションで**+**をクリック
3. グループを選択

### 方法4: API経由

動的グループの場合、APIがよく使われる方法です：

```bash
POST /api/v1/contacts
{
  "email": "new@example.com",
  "firstName": "太郎",
  "lastName": "山田",
  "groupId": "動的グループのuuid"
}
```

連絡先はグループにリンクされたアクティブなキャンペーンに自動的に登録されます。

## グループから連絡先を削除

### リストから

1. 該当するグループでフィルタリング
2. 削除する連絡先を選択
3. **アクション** > **グループから削除**をクリック
4. 確認

### 連絡先カードから

1. 連絡先のカードを開く
2. 「グループ」セクションでグループの横のXをクリック
3. 確認

> **情報**
> グループから連絡先を削除しても、FluenzRから削除されません。連絡先データベースに残ります。

## キャンペーンでのグループの使用

### グループ全体を追加

1. キャンペーンを作成または編集
2. **連絡先を追加**をクリック
3. **グループ**タブを選択
4. 希望のグループをチェック
5. 確認

### 動的グループをリンク

動的グループの場合、キャンペーンとのリンクが特に重要：

1. キャンペーンを作成し、動的グループを追加
2. **キャンペーンを開始**（実行中ステータス）
3. グループに追加されたすべての新しい連絡先が自動的に登録

### グループを除外

場合によっては、あるグループをターゲットしながら別のグループを除外したいことがあります：

1. メイングループを追加
2. **連絡先を除外**をクリック
3. 除外するグループを選択

例：「すべての見込み客」をターゲットしながら「現在の顧客」を除外

## キャンペーンでのグループビュー

キャンペーンの詳細ビューで、**概要**タブに表示：

- 連絡先グループとその統計
- グループごとの連絡先数
- グループごとの開封率とクリック率
- 動的グループの「動的」バッジ

グループをクリックすると、個々の連絡先とそのメトリクスを表示。

## ベストプラクティス

### 命名

明確な命名規則を使用：

| ✅ 良い | ❌ 悪い |
|---------|--------|
| LinkedInリード2024 | グループ1 |
| アクティブ顧客 | テスト |
| Fintech見込み客 | 太郎 |
| [DYN] インバウンドリード | 動的 |

> **ヒント**
> 動的グループには[DYN]のプレフィックスを付けて、簡単に識別できるようにしましょう。

### 構造

グループを論理的に整理：

```
ソース別:
├── [DYN] LinkedInインバウンド
├── [DYN] ウェブサイトフォーム
├── TechDays展示会2024
└── CSVインポート1月

業種別:
├── Fintech
├── SaaS
├── Eコマース
└── サービス

ステータス別:
├── [DYN] 新規見込み客
├── 絞り込み済み見込み客
├── アクティブ顧客
└── 離脱
```

### 各タイプの使用時期

| 状況 | 推奨タイプ |
|-----------|-----------------|
| 一回限りの連絡先インポート | 静的 |
| API経由の継続フロー | 動的 |
| キャプチャフォーム | 動的 |
| イベントリスト | 静的 |
| 継続的ナーチャリングキャンペーン | 動的 |
| ワンショットキャンペーン | 静的 |

### メンテナンス

- グループを定期的にレビュー
- 古いグループをアーカイブまたは削除
- 重複するグループをマージ
- 動的グループにリンクされたキャンペーンを確認

## グループへの一括アクション

グループのページから以下が可能：

| アクション | 説明 |
|--------|-------------|
| **エクスポート** | 連絡先をCSVでダウンロード |
| **キャンペーンに追加** | すべての連絡先を追加 |
| **タグを適用** | 全員にタグを追加 |
| **削除** | グループ内のすべての連絡先を削除 |

> **注意**
> グループから連絡先を削除すると、グループからだけでなくFluenzRから完全に削除されます。

## グループ統計

グループページで確認：

- 連絡先の総数
- アクティブ vs 非アクティブの連絡先
- 平均エンゲージメント率
- 最後に送信されたキャンペーン
- グループタイプ（静的/動的）
- リンクされたアクティブなキャンペーン数（動的グループの場合）
