# 連絡先をインポート

FluenzRでは、CSVファイルを使用して数百から数千の連絡先を数クリックでインポートできます。

## CSVファイルの準備

### 必要な形式

CSVファイルは以下のルールに従う必要があります：

- **エンコーディング**: UTF-8（日本語文字対応）
- **区切り文字**: カンマ（,）またはセミコロン（;）
- **最初の行**: 列ヘッダー

### サポートされている列

| CSV列 | FluenzRフィールド | 必須 |
|-------------|---------------|-------------|
| `email` | メール | ✅ |
| `firstName` | 名 | いいえ |
| `lastName` | 姓 | いいえ |
| `company` | 会社 | いいえ |
| `jobTitle` | 役職 | いいえ |
| `phone` | 電話 | いいえ |
| `website` | ウェブサイト | いいえ |
| `[カスタムフィールド]` | カスタムフィールド | いいえ |

### ファイル例

```csv
email,firstName,lastName,company,jobTitle
taro.yamada@example.com,太郎,山田,Acme Corp,営業部長
hanako.suzuki@startup.io,花子,鈴木,StartupIO,CEO
jiro.tanaka@tech.jp,次郎,田中,TechJapan,CTO
```

## インポートプロセス

### 1. インポートにアクセス

1. **連絡先**に移動
2. **CSVをインポート**をクリック

### 2. ファイルを選択

1. **ファイルを選択**をクリックするか、CSVをドラッグ＆ドロップ
2. FluenzRが自動的にファイルを分析

### 3. 列をマッピング

FluenzRは自動的に列を検出しますが、調整も可能：

1. 各CSV列が正しいFluenzRフィールドに対応していることを確認
2. 必要に応じてドロップダウンメニューで修正
3. 関連しない列は「無視」を選択

### 4. インポートオプション

| オプション | 説明 |
|--------|-------------|
| **宛先グループ** | すべての連絡先をグループに追加 |
| **タグ** | すべての連絡先にタグを適用 |
| **重複の処理** | 無視、更新、または作成 |

### 5. プレビューと検証

1. 最初の行のプレビューを確認
2. FluenzRが潜在的なエラー（無効なメールなど）を報告
3. 問題を修正するか、エラー行を無視することを承認

### 6. インポートを開始

**インポート**をクリックして開始。進行状況バーが表示されます。

## 重複の処理

メールが既に存在する場合、3つのオプションがあります：

| オプション | 動作 |
|--------|--------------|
| **無視** | 既存の連絡先は変更されない |
| **更新** | 連絡先のフィールドが新しい値で更新される |
| **作成** | ⚠️ 重複を作成（非推奨） |

> **情報**
重複検出はメールアドレスのみで行われます（大文字小文字を区別しない）。

## インポート後

### インポートレポート

インポート完了後、レポートに表示：

- インポートされた連絡先数
- 検出された重複数
- エラー数（詳細付き）

### 確認

1. **連絡先**に移動
2. インポート時に使用したグループまたはタグでフィルタリング
3. いくつかの連絡先をランダムに確認

## 制限とパフォーマンス

| プラン | インポートあたりの制限 |
|------|------------------|
| Free | 300件 |
| Starter | 10,000件 |
| Pro | 50,000件 |

> **注意**
大量インポート（10,000件以上）の場合、パフォーマンス向上のためファイルを複数のバッチに分割してください。

## トラブルシューティング

### "エンコーディングが不正"

症状：特殊文字が読めない

解決策：
1. ExcelまたはLibreOfficeでファイルを開く
2. 名前を付けて保存 > CSV UTF-8形式
3. 再インポート

### "メール列が検出されない"

解決策：
1. 最初の行にヘッダーがあることを確認
2. ヘッダーは正確に`email`（小文字）である必要があります
3. 必要に応じて手動でマッピング

### "エラーが多すぎる"

10%以上の行がエラーの場合：
1. エラーレポートをダウンロード
2. ソースファイルを修正
3. 修正した行のみを再インポート

### インポートがブロックされた

インポートがブロックされているように見える場合：
1. 数分待つ（大きなファイルは時間がかかります）
2. ページを更新
3. 連絡先に追加されているか確認
4. 問題が続く場合はサポートに連絡

## ベストプラクティス

### インポート前

- ファイルをクリーンアップ（空行を削除、エラーを修正）
- 可能であれば外部サービスでメールを検証
- 使用するグループとタグを準備

### インポート中

- インポート中はウィンドウを閉じない
- 確認のため予想される連絡先数をメモ

### インポート後

- インポートレポートを確認
- テストキャンペーンでいくつかの連絡先をテスト
- 元のCSVファイルのコピーを保管
