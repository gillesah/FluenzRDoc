# トラブルシューティング

このガイドでは、FluenzRで発生する最も一般的な問題の解決方法を説明します。

## メールアカウント

### 「Gmail接続に失敗」

**症状：** OAuth接続が失敗またはフリーズする。

**解決策：**
1. ポップアップがブロックされていないか確認
2. 広告ブロッカーを一時的に無効化
3. 別のブラウザを試す（Chrome推奨）
4. キャッシュとCookieをクリア
5. すべてのGoogleアカウントからログアウトして再接続

### Gmailで「検証されていないアプリ」

**症状：** 接続時にGoogleの警告が表示される。

**解決策：**
1. 「詳細設定」をクリック
2. 「FluenzRに移動（安全でない）」をクリック
3. Google認証は申請中です

### 「トークンの有効期限切れ」

**症状：** 送信時にエラーメッセージ、アカウントが「エラー」状態。

**解決策：**
1. メールアカウントに移動
2. アカウントの横にある「再接続」をクリック
3. OAuthプロセスを再度実行
4. 問題が続く場合は、アカウントを削除して再接続

### 「Gmailクォータ超過」

**症状：** メールが送信されない、クォータエラー。

**解決策：**
1. 24時間待つ（クォータはリセットされる）
2. 送信量を減らす
3. 追加のメールアカウントを追加
4. 手動で大量のメールを送信していないか確認

### 「SMTP接続に失敗」

**症状：** SMTPアカウントを接続できない。

**確認事項：**
1. サーバーとポートは正しいか？
2. ユーザー名 = 完全なメールアドレスか？
3. パスワードまたはアプリパスワードか？
4. 暗号化タイプ（TLS/SSL）は正しいか？
5. ファイアウォールが接続をブロックしていないか？

## キャンペーン

### 「キャンペーンがブロック / メールが送信されない」

**症状：** キャンペーンはアクティブだがメールが送信されない。

**確認事項：**
1. メールアカウントは接続されてアクティブか？
2. タイミング設定は正しいか？（曜日、時間）
3. キャンペーンに連絡先はいるか？
4. 連絡先は「pending」ステータスか？

**解決策：**
1. メールアカウントの状態を確認
2. 必要に応じてアカウントを再接続
3. 送信時間に現在の時刻が含まれているか確認

### 「連絡先がシーケンスでブロック」

**症状：** 連絡先がステップで止まっている。

**考えられる原因：**
1. 遅延がまだ経過していない
2. メールアカウントがエラー状態
3. 送信スケジュールが一致しない

**解決策：**
1. 設定された遅延を確認
2. キャンペーンのタイミング設定を確認
3. 必要に応じてメールアカウントを再接続

### 「変数が置換されない」

**症状：** `{{firstName}}`がそのままメールに表示される。

**解決策：**
1. 変数のスペルを確認
2. 手動入力ではなく変数メニューを使用
3. 連絡先にそのフィールドが存在するか確認
4. フォールバックを追加：`{{firstName|there}}`

### 「メールは送信されたがトラッキングされない」

**症状：** 「送信済み」カウンターがゼロのまま。

**原因：** 統計更新の遅延。

**解決策：** 数分待ってページを更新。

## 連絡先

### 「CSVインポートに失敗」

**症状：** インポート時にエラー、または不正なデータ。

**確認事項：**
1. UTF-8エンコーディングか？
2. 最初の行がヘッダーか？
3. 「email」列があるか？
4. 問題のある特殊文字がないか？

**解決策：**
1. CSVをExcel/LibreOfficeで開く
2. 「CSV UTF-8」として保存
3. email列が「email」という名前か確認
4. 空行を削除

### 「メールが既に存在」

**症状：** 連絡先を作成できない、メールが重複。

**解決策：** メールは一意である必要があります。既存の連絡先を編集するか、別のメールを使用してください。

### 「バウンスが多すぎる」

**症状：** バウンス率が高く、評判が低下。

**解決策：**
1. 連絡先リストをクリーンアップ
2. インポート前にメール検証サービスを使用
3. エラーの連絡先を削除
4. 収集元を確認

## 配信率

### 「メールがスパムに」

**症状：** メールが受信者のスパムフォルダに入る。

**即時の解決策：**
1. 送信量を減らす
2. ウォーミングを有効化（最低2-4週間）
3. コンテンツを簡素化（リンク減少、画像なし）
4. 各メールをパーソナライズ

**長期的な解決策：**
1. ドメインのSPF、DKIM、DMARCを設定
2. カスタムトラッキングドメインを使用（Pro）
3. 良好なリストハイジーンを維持
4. ウォーミングを常に有効に

### 「評判スコアが低下」

**症状：** メールアカウントのヘルス指標が低下。

**原因：**
- 急いで大量送信
- バウンス率が高い
- エンゲージメント（開封、返信）が少ない
- スパム苦情

**解決策：**
1. 一時的に送信量を減らす
2. ウォーミング強度を上げる
3. リストの品質を改善
4. コンテンツを見直す

### 「メールが配信されない（ソフトバウンス）」

**症状：** メールが「ソフトバウンス」とマーク。

**原因：** 一時的な問題（メールボックス満杯、サーバー利用不可）。

**対応：** FluenzRが自動的に再試行します。問題が続くと、連絡先はハードバウンスになります。

### 「ハードバウンス」

**症状：** メールが永続的にエラーとマーク。

**原因：** 無効または存在しないメールアドレス。

**対応：** これらの連絡先を削除またはアーカイブ。元のリストをクリーンアップ。

## インターフェース

### 「ページが読み込まれない」

**解決策：**
1. ページを更新（F5）
2. ブラウザキャッシュをクリア
3. シークレットモードで試す
4. 別のブラウザを試す
5. インターネット接続を確認

### 「変更が保存されない」

**解決策：**
1. インターネット接続を確認
2. 更新して再試行
3. 別のブラウザで試す
4. 問題が続く場合はサポートに連絡

### 「グラフ/統計が不正確」

**考えられる原因：** ブラウザキャッシュ。

**解決策：**
1. ページを更新
2. ブラウザキャッシュをクリア
3. 統計には数分の遅延がある場合があります

## Webhooks / API（Pro）

### 「Webhookが受信されない」

**確認事項：**
1. URLは公開アクセス可能か？
2. HTTPSが有効か？
3. 30秒以内に応答しているか？
4. 2xxコードを返しているか？

**デバッグ：**
1. 設定 > Webhooksでログを確認
2. 「テスト」ボタンを使用
3. サーバーのログを確認

### 「APIエラー401」

**原因：** APIキーが無効または欠落。

**解決策：**
1. キーがAuthorizationヘッダーに含まれているか確認
2. 正しい形式：`Bearer flz_sk_xxx`
3. 必要に応じて新しいキーを生成

### 「レート制限に達した」

**症状：** エラー429。

**解決策：**
1. リクエスト間隔を空ける
2. バックオフ付きのリトライシステムを実装
3. 呼び出しを最適化（バッチ、キャッシュ）
