# 配信率ガイド

配信率とは、メールが受信トレイ（スパムではなく）に到達する能力です。これはキャンペーン成功の鍵となる要素です。

## 基本

### 配信率の仕組み

送信されたすべてのメールは、プロバイダー（Gmail、Outlookなど）によっていくつかの基準で分析されます：

1. **送信者の評判**：アカウントの送信履歴
2. **認証**：SPF、DKIM、DMARC
3. **コンテンツ**：スパムワード、リンク、画像
4. **エンゲージメント**：受信者の開封率と返信率
5. **行動**：送信量と頻度

### 3つの可能な宛先

| 宛先 | 意味 | 影響 |
|-------------|---------------|--------|
| **受信トレイ** | 目標達成 | メールが見られ読まれる |
| **スパム** | 迷惑メールとしてフィルタリング | ほとんど見られない |
| **バウンス** | 配信されない | 受信されない |

## DNS設定

### SPF（Sender Policy Framework）

SPFは、どのサーバーがあなたのドメインからメールを送信できるかを示します。

**確認：**
```bash
dig TXT yourdomain.com
```

**標準設定：**
```
v=spf1 include:_spf.google.com ~all
```

### DKIM（DomainKeys Identified Mail）

DKIMは、メールに暗号署名を追加して真正性を証明します。

**設定：** メールプロバイダー経由（Google Workspace、Microsoft 365など）

### DMARC（Domain-based Message Authentication）

DMARCは、認証されていないメールをどう処理するかを受信者に指示します。

**標準設定：**
```
v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.com
```

### 設定の確認

無料ツールを使用：
- [mail-tester.com](https://www.mail-tester.com)
- [mxtoolbox.com](https://mxtoolbox.com)
- [dmarcian.com](https://dmarcian.com/dmarc-inspector/)

## 送信者の評判

### 評判とは？

プロバイダーが以下に基づいて付与するスコア：
- 送信履歴
- スパム苦情率
- バウンス率
- 受信者のエンゲージメント

### 評判の構築方法

1. **小規模から開始**：最初の数週間は20-30通/日
2. **徐々に増加**：問題なければ週+20%
3. **ウォーミングを有効化**：ポジティブなエンゲージメントをシミュレート
4. **品質を維持**：良好な開封率と返信率

### 健全性指標

| 指標 | 良好 | 注意 | 危険 |
|------------|-----|-----------|----------|
| バウンス率 | <2% | 2-5% | >5% |
| スパム苦情 | <0.1% | 0.1-0.5% | >0.5% |
| 開封率 | >25% | 15-25% | <15% |

## ウォーミング

### なぜアカウントをウォーミングするのか？

新規または使用頻度の低いメールアカウントには確立された評判がありません。ウォーミングは：
- ポジティブな活動を生成
- エンゲージメントシグナルを改善
- 徐々に評判を構築

### 仕組み

1. FluenzRがネットワーク内の他のアカウントにメールを送信
2. これらのメールは自動的に開封され返信される
3. 必要に応じてスパムから取り出される
4. これにより自然でポジティブなメール活動をシミュレート

### 推奨期間

| 状況 | ウォーミング期間 |
|-----------|------------------|
| 新規アカウント | プロスペクティング前2-4週間 |
| 使用頻度の低いアカウント | 1-2週間 |
| 配信率の問題 | 集中的に4-6週間 |

[ウォーミング完全ガイド](/docs/email-accounts/warming)

## コンテンツのベストプラクティス

### 避けるべきこと

| 要素 | 理由 |
|---------|----------|
| 大文字の乱用 | 明らかなスパムシグナル |
| 「無料」「緊急」「特別オファー」 | スパムトリガーワード |
| 多すぎるリンク（2以上） | 商業的シグナル |
| テキストなしの画像 | ニュースレター/広告シグナル |
| 大きな添付ファイル | 初回連絡では疑わしい |
| URL短縮サービス | スパマーがよく使用 |
| 複雑なHTML | マスメールシグナル |

### すべきこと

| 要素 | メリット |
|---------|----------|
| パーソナライゼーション | 本物のメッセージであることを証明 |
| シンプルなテキスト | 個人的なメールに見える |
| 最大1つのリンク | 控えめ |
| プロフェッショナルな署名 | 信頼性 |
| 質問 | 自然なエンゲージメント |
| 短く直接的 | 受信者への配慮 |

### 最適化されたメールの例

```
件名：{{firstName}}さん、簡単な質問

{{firstName}}様

{{company}}が{{sector}}分野で事業を
拡大しているのを拝見しました。

山田太郎
```

## リストハイジーン

### 定期的なクリーニング

| アクション | 頻度 |
|--------|-----------|
| ハードバウンスを削除 | 即時 |
| 購読解除を削除 | 自動 |
| 非エンゲージ（6ヶ月）をアーカイブ | 月次 |
| 新規メールを検証 | インポート時 |

### メール検証

リストをインポートする前に、検証サービスを使用：
- [Hunter.io](https://hunter.io)
- [NeverBounce](https://neverbounce.com)
- [ZeroBounce](https://zerobounce.net)

これにより特定：
- 無効なメール
- スパムトラップ
- 一時的/使い捨てメール

### 品質の高いソース

| ソース | 標準的な品質 |
|--------|-----------------|
| ウェブサイトのオプトイン | 優秀 |
| イベントフォーム | 良好 |
| LinkedIn（手動） | 良好 |
| リスト購入 | 非常に悪い ⚠️ |
| 無差別スクレイピング | 悪い |

## 送信量と頻度

### 段階的な増加

| 週 | 推奨送信量 |
|---------|-------------------|
| 1 | 20-30/日 |
| 2 | 40-50/日 |
| 3 | 60-80/日 |
| 4以降 | 100+/日 |

### 日中の分散

- 1時間で全て送信しない
- 日中に分散（FluenzRが自動的に実行）
- ピークを避ける

### フォローアップの頻度

| 間隔 | 印象 |
|-------|------------|
| 1日 | せっかち |
| 2-3日 | 許容範囲 |
| 4-5日 | 理想的 |
| 7日以上 | 勢いを失う |

## モニタリングと対応

### 監視すべき指標

| 指標 | 警告閾値 |
|----------|---------------|
| バウンス率 | >5% |
| 開封率 | <15%（低下中） |
| スパム苦情 | >0.1% |

### 問題が発生したら

**開封率の急激な低下：**
1. 即座に送信量を減らす
2. アカウントがブロックされていないか確認
3. ウォーミングを強化
4. メールコンテンツを見直す

**バウンスの急増：**
1. キャンペーンを停止
2. リストをクリーンアップ
3. 不正なメールの原因を特定
4. クリーンなリストで再開

**スパム苦情：**
1. そのセグメントへの送信を停止
2. ターゲティングを見直す
3. メッセージの関連性を改善
4. 連絡のコンテキストを明確に追加

## 配信率チェックリスト

キャンペーンを開始する前に：

- [ ] メールアカウントをウォーミング済み（2週間以上）
- [ ] DNS設定済み（SPF、DKIM、DMARC）
- [ ] リスト検証済み（バウンス<2%）
- [ ] パーソナライズされたコンテンツ
- [ ] スパムワードなし
- [ ] 最大1-2個のリンク
- [ ] 購読解除リンクあり
- [ ] 段階的な送信量を計画
- [ ] キャンペーン中もウォーミングを有効
