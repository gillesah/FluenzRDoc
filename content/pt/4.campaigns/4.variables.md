# Variáveis e personalização

As variáveis permitem personalizar automaticamente seus emails com informações de cada contato.

## Por que personalizar?

### Melhor taxa de abertura

Um assunto personalizado obtém 26% mais aberturas do que um assunto genérico.

### Melhor engajamento

Emails personalizados têm uma taxa de resposta 2x maior do que emails genéricos.

### Imagem profissional

A personalização mostra que você fez sua pesquisa e está se dirigindo à pessoa certa.

## Variáveis disponíveis

### Variáveis padrão

| Variável | Descrição | Exemplo |
|----------|-------------|---------|
| firstName | Nome | João |
| lastName | Sobrenome | Silva |
| email | Email | joao@example.com |
| company | Empresa | Acme Corp |
| jobTitle | Cargo | Diretor de Vendas |
| phone | Telefone | +55 11 98765-4321 |
| website | Site | https://acme.com |

### Variáveis do sistema

| Variável | Descrição |
|----------|-------------|
| unsubscribe_link | Link de descadastro |
| current_date | Data de hoje (formato local) |
| current_year | Ano atual |

### Campos personalizados

Seus campos personalizados estão disponíveis como variáveis:

| Variável | Exemplo |
|----------|---------|
| sector | Fintech |
| company_size | 50-100 |
| source | LinkedIn |

## Usando variáveis

### No assunto

Exemplo: `Olá firstName, pergunta rápida`

Resultado: "Olá João, pergunta rápida"

### No conteúdo

```text
Olá [firstName],

Percebi que [company] está expandindo no setor [sector].
Como [jobTitle], você deve estar enfrentando...
```

Substitua os colchetes pela sintaxe de variável com chaves duplas.

### Menu de inserção

1. No editor de email, posicione seu cursor
2. Clique em **Variáveis** na barra de ferramentas
3. Selecione a variável para inserir

## Valores padrão (fallback)

Se uma variável estiver vazia para um contato, você pode definir um valor padrão com a sintaxe: `variable|default_value`

### Exemplos

- `firstName|você` → "João" ou "você" se o nome estiver vazio
- `company|sua empresa` → "Acme Corp" ou "sua empresa" se vazio

## Melhores práticas

### Sempre forneça um fallback

Nunca deixe uma variável sem um fallback possível para evitar emails com espaços em branco.

### Verifique a qualidade dos dados

Antes de usar uma variável, certifique-se de que:
- O campo está preenchido para a maioria dos contatos
- Os dados estão limpos (capitalização correta, etc.)

### Personalização natural

Evite personalização excessiva que pareça artificial. 1-2 variáveis por parágrafo é suficiente.

### Teste antes de enviar

1. Use a visualização com diferentes contatos
2. Envie um email de teste para você mesmo
3. Verifique casos onde dados estão faltando

## Personalização avançada (Plano Pro)

### Variáveis condicionais

Exiba texto baseado em uma condição. Por exemplo, exiba texto diferente dependendo se o cargo está preenchido ou não.

### Formatação

Opções de formatação disponíveis:
- uppercase: Converte para maiúsculas
- lowercase: Converte para minúsculas
- short: Formato curto para datas

## Exemplo de template

```text
Assunto: [firstName], pergunta sobre [company]

Olá [firstName],

Estou entrando em contato porque [company] parece se encaixar
perfeitamente no perfil das empresas que apoiamos.

Você estaria disponível para uma conversa de 15 minutos?

Atenciosamente,
[Assinatura]

[unsubscribe_link]
```

Substitua os colchetes pela sintaxe de variável com chaves duplas.

## Solução de problemas

### Variável exibida como está

Se a variável é exibida em vez do valor:
- Verifique a ortografia da variável
- Verifique se o campo existe para o contato
- Use o menu de variáveis em vez de entrada manual

### Dados incorretos

Se "joão" é exibido em vez de "João":
- Limpe seus dados na origem
- Use importação CSV com dados limpos
- Corrija manualmente contatos importantes

### Variável vazia

Se nada é exibido:
- Adicione um fallback
- Verifique se o campo não está vazio para este contato
