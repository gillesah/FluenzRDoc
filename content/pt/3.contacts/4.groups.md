# Grupos de Contatos

Os grupos permitem organizar seus contatos e segmentar com precisão suas campanhas. O FluenzR oferece dois tipos de grupos: **estáticos** e **dinâmicos**.

## Por que usar grupos?

### Organização

- Classifique seus contatos por origem (LinkedIn, Feiras, Site)
- Separe por indústria ou tamanho da empresa
- Distinga clientes de prospects

### Segmentação

- Crie campanhas específicas por segmento
- Personalize suas mensagens de acordo com o grupo
- Meça o desempenho por segmento

### Gestão

- Exporte facilmente um segmento
- Aplique ações em massa
- Acompanhe a evolução de cada grupo

## Tipos de Grupos

O FluenzR oferece dois tipos de grupos para atender diferentes necessidades:

### Grupos Estáticos

Os grupos estáticos são o método tradicional de gerenciamento de contatos:

| Característica | Descrição |
|----------------|-----------|
| **Adição manual** | Os contatos são adicionados manualmente ou por importação |
| **Persistência** | Um contato permanece no grupo até ser removido explicitamente |
| **Controle total** | Você decide exatamente quem faz parte do grupo |
| **Casos de uso** | Listas fixas, importações CSV, segmentos pontuais |

### Grupos Dinâmicos

Os grupos dinâmicos permitem a inscrição automática em campanhas:

| Característica | Descrição |
|----------------|-----------|
| **Inscrição automática** | Novos contatos automaticamente inscritos em campanhas ativas |
| **Tempo real** | Assim que um contato é adicionado ao grupo, ele entra nas campanhas vinculadas |
| **Escalável** | Perfeito para fluxos contínuos (API, formulários, enriquecimento) |
| **Casos de uso** | Leads inbound, workflows automatizados, campanhas contínuas |

> **Dica**
> Use grupos dinâmicos para campanhas de nurturing contínuo onde novos contatos chegam regularmente.

## Criar um Grupo

### 1. Acessar a Criação

1. Vá para **Contatos**
2. Clique em **Grupos** na barra lateral
3. Clique em **+ Novo Grupo**

### 2. Configurar o Grupo

| Campo | Descrição | Exemplo |
|-------|-----------|---------|
| **Nome** | Identificador do grupo | "Leads LinkedIn Q4" |
| **Descrição** | Nota explicativa (opcional) | "Contatos coletados via LinkedIn no Q4 2024" |
| **Tipo** | Estático ou Dinâmico | Dinâmico |
| **Cor** | Cor de identificação | Azul |

### 3. Escolher o Tipo

#### Grupo Estático
Selecione **Estático** se:
- Você está importando uma lista fixa de contatos
- Você quer controle manual total
- A lista não mudará frequentemente

#### Grupo Dinâmico
Selecione **Dinâmico** se:
- Novos contatos chegam regularmente (API, formulários)
- Você quer que eles sejam automaticamente inscritos em campanhas
- Você tem campanhas de nurturing contínuo

### 4. Salvar

Clique em **Criar** para finalizar.

## Funcionamento dos Grupos Dinâmicos

### Inscrição Automática em Campanhas

Quando um contato é adicionado a um grupo **dinâmico**:

1. O FluenzR verifica se há campanhas ativas vinculadas a este grupo
2. Se houver, o contato é automaticamente inscrito nessas campanhas
3. O workflow da campanha começa imediatamente para este contato

```
Novo contato → Grupo dinâmico → Campanha ativa → Início do workflow
```

### Condições Necessárias

Para que a inscrição automática funcione, o contato deve:

- ✅ Ter um endereço de email válido
- ✅ Estar em um grupo do tipo **DINÂMICO**
- ✅ Ter status **ATIVO**
- ✅ Não estar já inscrito na campanha

### Exemplo de Workflow

1. Você cria um grupo dinâmico "Leads Inbound"
2. Você cria uma campanha de nurturing e a associa a este grupo
3. Você lança a campanha
4. Cada novo lead adicionado ao grupo (via API, importação, formulário):
   - É automaticamente inscrito na campanha
   - Recebe o primeiro email do workflow
   - Progride na sequência

> **Aviso**
> Um grupo dinâmico pode estar vinculado a múltiplas campanhas ativas. Cada novo contato será inscrito em **todas** as campanhas ativas vinculadas ao grupo.

## Adicionar Contatos a um Grupo

### Método 1: Na Criação

Ao criar ou importar contatos, selecione o grupo de destino.

### Método 2: Da Lista de Contatos

1. Selecione contatos (caixas de seleção)
2. Clique em **Ações** > **Adicionar ao grupo**
3. Selecione o grupo
4. Confirme

### Método 3: Da Ficha de Contato

1. Abra a ficha de um contato
2. Na seção "Grupos", clique em **+**
3. Selecione o grupo

### Método 4: Via API

Para grupos dinâmicos, a API é frequentemente o método preferido:

```bash
POST /api/v1/contacts
{
  "email": "novo@example.com",
  "firstName": "João",
  "lastName": "Silva",
  "groupId": "uuid-do-grupo-dinâmico"
}
```

O contato será automaticamente inscrito nas campanhas ativas vinculadas ao grupo.

## Remover Contatos de um Grupo

### Da Lista

1. Filtre pelo grupo relevante
2. Selecione os contatos a remover
3. Clique em **Ações** > **Remover do grupo**
4. Confirme

### Da Ficha de Contato

1. Abra a ficha do contato
2. Na seção "Grupos", clique no X ao lado do grupo
3. Confirme

> **Info**
> Remover um contato de um grupo não o exclui do FluenzR. Ele permanece na sua base de contatos.

## Usar Grupos em Campanhas

### Adicionar um Grupo Completo

1. Crie ou edite uma campanha
2. Clique em **Adicionar contatos**
3. Selecione a aba **Grupos**
4. Marque o(s) grupo(s) desejado(s)
5. Confirme

### Vincular um Grupo Dinâmico

Para grupos dinâmicos, o vínculo com a campanha é particularmente importante:

1. Crie uma campanha e adicione o grupo dinâmico
2. **Lance a campanha** (status RUNNING)
3. Todos os novos contatos adicionados ao grupo serão automaticamente inscritos

### Excluir um Grupo

Em alguns casos, você quer segmentar um grupo mas excluir outro:

1. Adicione o grupo principal
2. Clique em **Excluir contatos**
3. Selecione o grupo a excluir

Exemplo: Segmentar "Todos os prospects" mas excluir "Clientes atuais"

## Visão de Grupos em Campanhas

Na visão de detalhes de uma campanha, a aba **Resumo** mostra:

- Grupos de contatos com suas estatísticas
- Número de contatos por grupo
- Taxas de abertura e clique por grupo
- Badge "Dinâmico" para grupos dinâmicos

Clique em um grupo para ver os contatos individuais e suas métricas.

## Melhores Práticas

### Nomenclatura

Use convenções de nomenclatura claras:

| ✅ Bom | ❌ Ruim |
|--------|---------|
| Leads LinkedIn 2024 | Grupo 1 |
| Clientes Ativos | Teste |
| Prospects Fintech | João |
| [DYN] Leads Inbound | Dinâmico |

> **Dica**
> Prefixe seus grupos dinâmicos com [DYN] para identificá-los facilmente.

### Estrutura

Organize seus grupos logicamente:

```
Por origem:
├── [DYN] LinkedIn Inbound
├── [DYN] Formulário Web
├── Feira TechDays 2024
└── Importação CSV Janeiro

Por setor:
├── Fintech
├── SaaS
├── E-commerce
└── Serviços

Por status:
├── [DYN] Novos Prospects
├── Prospects Qualificados
├── Clientes Ativos
└── Churn
```

### Quando Usar Cada Tipo

| Situação | Tipo Recomendado |
|----------|------------------|
| Importação pontual de contatos | Estático |
| Fluxo contínuo via API | Dinâmico |
| Formulário de captação | Dinâmico |
| Lista de evento | Estático |
| Campanha de nurturing contínuo | Dinâmico |
| Campanha pontual | Estático |

### Manutenção

- Revise seus grupos regularmente
- Arquive ou exclua grupos obsoletos
- Mescle grupos redundantes
- Verifique as campanhas vinculadas a grupos dinâmicos

## Ações em Massa em um Grupo

Da página de um grupo, você pode:

| Ação | Descrição |
|------|-----------|
| **Exportar** | Baixar contatos em CSV |
| **Adicionar à campanha** | Adicionar todos os contatos |
| **Aplicar tags** | Adicionar tags a todos |
| **Excluir** | Excluir todos os contatos do grupo |

> **Aviso**
> Excluir contatos de um grupo os remove permanentemente do FluenzR, não apenas do grupo.

## Estatísticas de Grupo

Na página do grupo, visualize:

- Número total de contatos
- Contatos ativos vs inativos
- Taxa de engajamento média
- Última campanha enviada
- Tipo de grupo (Estático/Dinâmico)
- Número de campanhas ativas vinculadas (para grupos dinâmicos)
